# 版本记录

本文件只记录 Git release / tag 级别的版本变化。

运行策略版本、工作流分支、脚本实现版本不要写成 release 版本。详细规则见：

- `docs/版本管理规范.md`
- `config/version-manifest.json`

## 当前状态

- 当前已存在 Git tag：`v1.0.0`、`v1.1.0`
- `main` 当前不是一个已打 tag 的 release
- 下一次整理发布建议使用：`v1.1.1`

## Unreleased

- 新增版本管理规范和版本矩阵。
- 移除早期 RunComfy Serverless / Node CLI 实验入口，保留当前 Vast 主线需要的 Node 工作流转换脚本。
- 明确区分四类版本：
  - Git release：`v1.0.0`、`v1.1.0`
  - 运行策略：`1.0-cold`、`1.1-machine-registry`
  - 工作流分支：`wan_2_2_animate`、`wan_2_2_animate_segmented`
  - 脚本实现：`segmented v1/v2/v3`
- 纠正历史口径：`v1.1.1` 以前只写在本文档中，但仓库尚无对应 tag，因此不能算正式 release。

## v1.1.0 - Wan2.2 Vast workflow 机器库生产路径

- 新增机器库选择路径。
- 支持 Vast.ai 实例的机器注册表管理。
- 支持 `1.1-machine-registry`：命中历史成功机器时启用 `WarmStart`，未命中则按冷启动处理。
- 优化 Wan2.2 动画工作流的生产环境部署。

## v1.0.0 - Wan2.2 Vast workflow 冷启动基线

- 建立 Wan2.2 Vast 工作流基线版本。
- 支持短视频自动化重写与 AI 视频生成管道。
- 集成 RunComfy 云端推理、OpenAI 转写/重写、Cloudflare R2 存储。
- 支持多种 AI 模型：Wan Animate、LivePortrait、LatentSync、MultiTalk、DreamID 等。
- CLI 提供 `inspect`、`transcribe`、`rewrite`、`pipeline`、`submit` 等完整命令链。
